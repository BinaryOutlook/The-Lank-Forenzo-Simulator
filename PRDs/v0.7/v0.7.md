# V0.7 PRD: Player Tutorial Route

Status: implemented for issue #65  
Owner: BinaryOutlook  
Last updated: 2026-05-12  
Iteration type: Player onboarding, UI route, and documentation

## Summary

`V0.7` adds a first-class Tutorial page for new players who need a clear starting point before entering a run. The page explains the premise, objective, controls, practical round loop, important simulation concepts, run UI areas, and first-run tips without interrupting active play.

No `v0.7-demo.html` is included because this iteration extends the existing shell/page design language instead of introducing a new visual direction.

## Goals

1. Add a stable `/tutorial` route that supports direct navigation and browser reloads under the app router.
2. Add a top-navigation Tutorial link beside About while preserving About, Options, and run navigation behavior.
3. Keep tutorial copy concise enough to scan, but specific enough for a first-time player to begin a run confidently.
4. Cover the objective, controls/interaction model, run flow, key concepts, UI areas, and tips.
5. Use the existing theme tokens, responsive layout patterns, and interaction feedback controls.
6. Add coverage for the page content and primary navigation entry.
7. Update durable route/navigation documentation.

## Non-Goals

- Do not change simulation rules, balance, content packs, save format, endings, or run resolution.
- Do not replace the About page; Tutorial teaches play, while About explains product intent and satire.
- Do not add a guided in-run overlay or blocking first-run modal.
- Do not introduce new dependencies or a separate mobile-only layout.

## Implementation Notes

- The Tutorial page lives under `src/screens/tutorial/` as a dedicated screen component and CSS module.
- The route is registered as `/tutorial` in `src/app/router.tsx`.
- The app shell primary navigation now renders About, Tutorial, and Options in that order.
- The Tutorial page can start a new run when no active run exists, or return directly to `/run` when an active run is present.
- The page documents the split between company health and personal success, faction memory, dossier/scandal pressure, hazards, strategic reserves, endings, and ending recaps.
