V0.2 demo reference

More event variety, clearer stat semantics, tighter consequence recap.

This reference captures the direction for the `V0.2` packet: the event system scales beyond the tiny current pool, indicator meaning is based on reviewed metric semantics instead of raw sign, and the consequence feed becomes a focused latest-fallout rail.

Event scale

The current baseline of `18` events is not enough to carry repeated runs. The content structure should support a much larger authored pool and broader category coverage.

Current: 18 total events
Target structure: category packs or maintainable multi-file authoring
Long-term target: roughly 30x to 50x current volume

Metric semantics

Indicator color and ranking logic should follow reviewed metric direction, not the default idea that all positive values are good and all negative values are bad.

Safety integrity +8
Legal heat -6
Public anger -4
Workforce size -500 may be contextual

Consequence feed

The right rail should read like a recent chain of fallout, not a full archive. Show only the newest `10` entries on the run screen.

R7 Board RevoltDirectors are losing faith faster than counsel can calm them.
R7 Quarter CloseOperations bleed cash faster than the story can hide it.
R6 Tabloid ProfileThe image campaign turns into a catalogue of executive rot.
Visible rail cap: 10 entries
  • Lower-is-better pressure metrics should no longer be treated as backwards danger signals.
  • Event authoring should be restructured before the content library grows into an unmaintainable single file.
  • The right rail should optimize for fast recap, not long-form historical storage.
