## Summary

`V0.6` is a simulation, diagnostics, and tuning pass for **The Lank Forenzo Simulator**. It does not chase raw content volume. It repairs the visible gaps where authored content, strategic endings, and bounded scripted play were failing to meet each other.

The useful quality target is:

$$
\text{V0.6 Quality} =
\text{Ending Viability}
\times
\text{Content Reachability}
\times
\text{Diagnostic Clarity}
$$

If any factor stays near zero, the authored game collapses into too narrow a possibility space.

No `v0.6-demo.html` is included in this packet because this iteration has no meaningful visual-design or HTML demo reference.

## Product Decision

The stable product thesis remains unchanged:

**The company and the player are not the same entity.**

`V0.6` protects that thesis by making the existing strategic exits and failures more dependable as play spaces:

- `Extraction` should remain powerful, but it must not be the only successful lane scripted play can exploit.
- `Merger` should reward a disciplined credibility-preservation sequence.
- `Bahamas` should reward early private-wealth and offshore-readiness planning.
- `Forced Removal` and `Prison` should appear often enough in reports to prove that greed, neglect, and exposure can punish the player.
- Low-reachability packs should either become visible under named archetypes or carry an explicit design note explaining why they remain rare.

## Goals

`V0.6` should:

1. Make all five endings reachable in either the default archetype matrix or bounded reachability report.
2. Add actionable diagnostics so balance failures are easier to attribute.
3. Increase decision coverage enough that authored packs are being exercised, not merely stored.
4. Explain reachability misses by failure mode: requirement never met, card never offered, card offered but not selected, delayed chain failed to resolve, ending threshold missed, or resource cost blocked the lane.
5. Add low-reachability archetype support for `safetyDenial` and `shadowSubsidiaries`.
6. Preserve deterministic simulation behavior for the same seed, content hash, and selected decision sequence.
7. Keep tuning grounded in the existing player fantasy instead of adding unrelated content volume.
8. Leave final report outputs and implementation notes in this PRD's change log.

## Non-Goals

`V0.6` does not:

- implement a new major UI surface
- add a backend, accounts, cloud saves, telemetry, or live service behavior
- replace the existing React, TypeScript, Vite, Zustand, Zod, Vitest, or Playwright stack
- expand into detailed route planning or dispatcher simulation
- add large volumes of content without proving existing content can surface
- make all endings equally likely or symmetrical
- remove extraction as a viable strategy
- loosen deterministic tests just to make balance tuning easier

## Implementation Guardrails

- Keep simulation deterministic for a fixed seed and content hash.
- Keep report code runnable outside React.
- Prefer typed content metadata over decision-ID substring matching.
- Do not use random surfacing boosts that cannot be explained by report diagnostics.
- Preserve exit-card availability when requirements are truly met.
- Treat strategic-resource costs as part of reachability; a card that is always unaffordable is effectively dead.
- Update content docs when pack purpose or rarity changes.
- Append a short implementation note to this PRD after significant balance passes.

## Blockers Diagnosed

### Bahamas

The matrix offshore bot could accumulate personal wealth, but it did not reliably cross the old Nassau readiness gate before heat and delayed offshore fallout eroded the path. In the baseline, representative offshore runs ended active after 24 rounds with enough wealth but only about `45` offshore readiness against a `60` requirement.

After improving strategic setup surfacing, the offshore bot also needed clearer lane discipline: newly surfaced merger and shell setup cards were attractive on raw impact score, so the offshore policy now explicitly prefers `bahamas` and `escape` tags while penalizing `merger` and `shell` distractions.

### Merger

The merger bot selected merger-flavored setup, especially `pre_merger_labor_cleanup`, but the selected delayed chain resolved to `route_overlap_chart`, which did not set `mergerOffer`. The `accept_merger_offer` exit therefore never appeared in matrix runs.

The fix keeps the setup sequence intact but changes that cleanup fallout to be capable of producing `merger_interest`, giving the bot a deterministic offer window without removing merger counter-pressure.

### Reachability Explorer

The reachability abstraction bucketed all personal wealth below `80` as one state, which hid the difference between being broke and being nearly liquid enough to flee. Offshore readiness also used generic pressure buckets. The explorer now has exit-aligned buckets for personal wealth and offshore readiness, so promising Nassau states survive frontier pruning.

## Implemented Changes

- Added `safety-denial` and `shadow-subsidiary` archetype bots to target the two baseline low-reachability packs.
- Tuned the offshore archetype toward Bahamas readiness and away from merger/shell distractions.
- Raised tray metadata bonuses for flag-setting and flagged follow-up decisions so authored strategic chains surface more consistently.
- Lowered `run_for_nassau` from `offshoreReadiness >= 60` and `personalWealth >= 55` to `offshoreReadiness >= 35` and `personalWealth >= 45`.
- Updated the missed Nassau recap window to match the new liquidity/readiness gate.
- Changed `pre_merger_labor_cleanup` fallout so the merger lane can produce `merger_interest` and therefore `mergerOffer`.
- Updated balance/reachability report labels to v0.6.
- Added regression tests for deterministic v0.6 archetypes, multiple successful matrix exits, and 4/5 reachability endings.
