# V0.6 PRD: Balance and Reachability Repair

Status: implemented in issue #18  
Parent roadmap: #17  
Source report: `docs/FUTURE_REPORT.md`

## Goal

Broaden the reachable play space before larger systems work. The v0.6 pass keeps the deterministic card/run model intact while repairing the most visible reachability gaps:

- successful exits should not be extraction-only in scripted matrix runs
- Bahamas and merger should have legible blockers and repaired paths
- low-confidence packs should receive targeted scripted coverage
- fixed seeds must still produce fixed reports

The useful balance target is:

$$
\text{Reachable Play Space} =
\text{viable exits} + \text{covered packs} + \text{diagnosable failures}
$$

## Blockers Diagnosed

### Bahamas

The matrix offshore bot could accumulate personal wealth, but it did not reliably cross the old Nassau readiness gate before heat and delayed offshore fallout eroded the path. In the baseline, representative offshore runs ended active after 24 rounds with enough wealth but only about `45` offshore readiness against a `60` requirement.

After improving strategic setup surfacing, the offshore bot also needed clearer lane discipline: newly surfaced merger and shell setup cards were attractive on raw impact score, so the offshore policy now explicitly prefers `bahamas`/`escape` tags and penalizes `merger`/`shell` distractions.

### Merger

The merger bot selected merger-flavored setup, especially `pre_merger_labor_cleanup`, but the selected delayed chain resolved to `route_overlap_chart`, which did not set `mergerOffer`. The `accept_merger_offer` exit therefore never appeared in matrix runs.

The fix keeps the setup sequence intact but changes that cleanup fallout to be capable of producing `merger_interest`, giving the bot a deterministic offer window without removing merger counter-pressure.

### Reachability Explorer

The reachability abstraction bucketed all personal wealth below `80` as one state, which hid the difference between being broke and being nearly liquid enough to flee. Offshore readiness also used generic pressure buckets. The explorer now has exit-aligned buckets for personal wealth and offshore readiness, so promising Nassau states survive frontier pruning.

## Implemented Changes

- Added `safety-denial` and `shadow-subsidiary` archetype bots to target the two baseline low-reachability packs.
- Tuned the offshore archetype toward Bahamas readiness and away from merger/shell distractions.
- Raised tray metadata bonuses for flag-setting and flagged follow-up decisions so authored strategic chains surface more consistently.
- Lowered `run_for_nassau` from `offshoreReadiness >= 60` and `personalWealth >= 55` to `offshoreReadiness >= 35` and `personalWealth >= 45`.
- Updated the missed Nassau recap window to match the new liquidity/readiness gate.
- Changed `pre_merger_labor_cleanup` fallout so the merger lane can produce `merger_interest` and therefore `mergerOffer`.
- Updated balance/reachability report labels to v0.6.
- Added regression tests for deterministic v0.6 archetypes, multiple successful matrix exits, and 4/5 reachability endings.

## Remaining Balance Notes

- `marketTheater` is the only remaining low-reachability pack in the default matrix. It is covered by the reachability report, but a future market/analyst archetype or tray-composer V2 pass should make that lane more robust under scripted bots.
- Forced removal is reachable in the reachability report but not the default matrix. This is acceptable for v0.6 because prison now supplies a matrix failure path and forced removal remains bounded-reachable.
- Delayed matrix event coverage is slightly lower because several repaired successful exits now resolve earlier; total triggered event coverage improved materially.
