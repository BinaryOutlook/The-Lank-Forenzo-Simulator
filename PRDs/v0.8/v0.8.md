# V0.8 PRD: Dedicated Decision Selection View

Status: implemented for issue #70  
Owner: BinaryOutlook  
Last updated: 2026-05-12  
Iteration type: Gameplay UI, responsive decision comparison, accessibility

## Summary

`V0.8` gives the five round decisions their own selection phase at `/run/decisions`. The existing run board stays intact, but players can expand the crowded decision tray into a roomy comparison view with persistent selected-count controls, readable reserve costs, consequence chips, and direct card selection.

No `v0.8-demo.html` is included because this iteration extends the established grim ledger visual system rather than introducing a separate art direction.

## Goals

1. Add a dedicated decision-selection route reachable from the active run screen without changing the stable top navigation order.
2. Show all currently available round decisions with enough room for category, tags, title, summary, strategic reserve cost, and consequence previews.
3. Keep `0/2 selected` style progress and reserve-spend summary visible while players compare options.
4. Preserve card click/touch selection, deselection, keyboard focus, and `aria-pressed` state.
5. Make selected, unselected, and unavailable card states visually distinct beyond color alone.
6. Use responsive card grids and sticky controls instead of fixed desktop-only compression.

## Non-Goals

- Do not change decision curation, simulation resolution, save format, resource-cost rules, or ending logic.
- Do not add a new top-level primary navigation item; `Run`, `About`, `Tutorial`, and `Options` remain stable.
- Do not remove the embedded run-screen decision tray while parallel navigation and tutorial work is still landing.
- Do not introduce a mobile-only gameplay rule set.

## Implementation Notes

- The dedicated screen lives under `src/screens/decision-selection/` and is registered as `/run/decisions`.
- The run-screen decision tray links to the dedicated view with an explicit action.
- `DecisionTray` now supports an expanded `selection` surface that reuses the same card selection logic while changing layout density, card spacing, and control behavior.
- The expanded surface keeps the quarter controls sticky: side-pinned on wide viewports and bottom-pinned on narrow viewports.
- Decision cards now include an explicit `Available`, `Queued`, or `Unavailable` badge so selection state is visible without relying on border color alone.
